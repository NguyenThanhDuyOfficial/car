## About The Project

[![Product Screen Shot][product-screenshot]](product-url)

**Car Rental Platform** is a full-stack web application that allows users to browse available cars, select rental dates, and complete orders seamlessly. The platform provides a smooth user experience with robust authentication and role-based access control.

**Key Features:**
- 🚗 Browse and filter available cars
- 📅 Select rental dates and durations
- ✅ Complete orders with payment integration
- 🔐 User authentication with JWT and role-based access (Admin/User)
- 📊 Full CRUD operations for cars, orders, users, and payments
- 📝 API documentation with Swagger

> **⚠️ Note:** This project is currently **paused**. Instead of focusing on a demo project, I want to dedicate more time to community-driven projects that create real impact. The project is nearly complete, with only the car filtering frontend component and UI/UX improvements remaining.

## Getting Started

To get a local copy up and running, follow these steps.
### Prerequisites

* **pnpm**
  ```sh
  npm install -g pnpm
  ```

* [OPTIONAL] **Docker**: Make sure you have **Docker** installed and running on your system.
  ```sh
  docker --version
  docker compose version
  ```

### One-Line Command

```sh
npm install -g pnpm &&
git clone https://github.com/NguyenThanhDuyOfficial/car.git &&
cd car &&
pnpm install &&
cp api/.env.example api/.env &&
cp web/.env.example web/.env &&
cd api && docker compose up -d &&
npx prisma generate &&
npx prisma migrate dev --name init &&
cd .. &&
pnpm dev
```

> **📌 Note:** After running `pnpm dev`, the application will start. You can then access:
> - **Frontend:** http://localhost:3000
> - **API Docs (Swagger):** http://localhost:3001/api/docs



### Installation

1. **Clone the repository**
   ```sh
   git clone https://github.com/NguyenThanhDuyOfficial/car.git
   cd car
   ```

2. **Install dependencies**
   ```sh
   pnpm install
   ```

3. **Set up environment variables**
   ```sh
   # Backend
   cp api/.env.example api/.env
   
   # Frontend
   cp web/.env.example web/.env
   ```

4. **[Optional] Run local database with Docker**
   ```sh
   cd api && docker compose up -d
   ```
5. Generate Prisma
    ```sh
    npx prisma generate
    npx prisma migrate dev --name init
    ```

6. **Start the development server**
   ```sh
   pnpm dev
   ```

   The application will be available at:
   - **Frontend:** [http://localhost:3000](http://localhost:3000)
   - **Backend API:** [http://localhost:3001/api/v1](http://localhost:3001/api/v1)
   - **API Documentation (Swagger):** [http://localhost:3001/api/docs](http://localhost:3001/api/docs)
